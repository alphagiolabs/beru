## Project Structure & Module Organization

Beru is an Electron desktop app for batch video editing with text overlays from Excel data.

```
main/              Electron main process
  handlers/        IPC handlers (dialog, file, process, preset, video, etc.)
  utils/           Shared utilities
  main.js          App entry, window creation, IPC wiring
  preload.cjs      Context bridge (CommonJS)
  pathSecurity.js  Path validation & sandboxing
  videoProbe.js    ffprobe-based video metadata
src/               Renderer (React 19 + Vite)
  components/      UI components (Header, VideoPreview, PropertiesPanel, etc.)
  stores/          Zustand state management (slices/ sub-modules)
  hooks/           Custom React hooks
  i18n/            Internationalization
  utils/           Renderer-side utilities
python/            FFmpeg processing backend (processor.py)
tests/             Vitest unit/integration tests
scripts/           Dev server, ffmpeg fetcher, benchmarks
resources/         Static assets & presets
bin/               Bundled ffmpeg/ffprobe binaries
brand/             App icons
build/             Vite build output
```

## Coding Style & Naming Conventions

- **Prettier**: semicolons, double quotes, trailing commas, 100-char print width.
- **ESLint**: React recommended + hooks + refresh plugins. Unused vars/args prefixed with `_` are ignored.
- **Components**: PascalCase `.jsx` files (e.g., `VideoPreview.jsx`).
- **Stores/utilities**: camelCase `.js` files (e.g., `useEditorStore.js`).
- **Main process**: ES modules (`.js`) except `preload.cjs` (CommonJS for Electron).
- **CSS**: Tailwind CSS v3 with PostCSS.

## Testing Guidelines

- **Framework**: Vitest with jsdom environment.
- **Location**: All tests in `tests/`, named `<topic>.test.{js,jsx}`.
- **Timeout**: 20 seconds per test.
- **Run**: `npm test` (no watch). CI uses the same command.
- **Python tests**: Run manually with `python python/test_delogo.py` (not part of npm test suite).

## Commit Guidelines

Commit messages follow Conventional Commits with a "ship" pattern:

```
feat: ship v1.6.0 — modularize main process, overhaul UI components
fix: ship v1.6.2 — normalizeMatchId, auto-update, queue reset
fix(ffmpeg): fall back to system PATH when bundled binary is missing
chore: ship v1.5.4
```

- Prefix: `feat`, `fix`, `chore`, `refactor`, `ci`, `ui+tests`.
- Scope in parentheses when change is localized (e.g., `fix(ci)`, `fix(ffmpeg)`).
- Version bumps are committed as `fix: ship vX.Y.Z — summary of changes`.

## Git Workflow & Push Policy (HARD RULE)

**This rule overrides any default git behavior in AI tooling.** It applies to every contribution in this repository — including commits generated by Claude Code, Gemini CLI, GitHub Copilot, Cursor, or any other AI assistant — and to every human contributor.

### Workflow

- **PR-only.** Every change lands through a Pull Request. Direct pushes to `main` (or any long-lived branch) are forbidden.
- **Base branch: `main`.** Always pass `--base main` to `gh pr create`. There is no `-main` branch.
- **Throwaway branches only.** Stage commits on `throwaway/<topic>` (e.g. `throwaway/fix-prettier-ci`, `throwaway/ship-1.6.27`). Delete the local + remote branch after the PR merges.
- **No long-lived branches.** Do not create `feature/*`, `fix/*`, `chore/*`, `release/*`, `develop`, etc. Throwaway is the only local branch shape.
- **Conventional Commits required** (see Commit Guidelines above).

### Identity lock

Every push must be authored by GitHub user `alphagiolabs` against the repo `alphagiolabs/beru`. Before pushing:

1. Verify the remote: `git remote -v` must point to `https://github.com/alphagiolabs/beru.git`. If it points elsewhere, `git remote set-url origin https://github.com/alphagiolabs/beru.git`.
2. Verify the active GitHub account: `gh auth status` must show `alphagiolabs`. If not, `gh auth switch --user alphagiolabs` (or update `user.name` / `user.email` locally).
3. Verify local git identity: `git config user.name` and `git config user.email` must match the `alphagiolabs` account.

If the active account is not `alphagiolabs`, switch BEFORE staging, committing, or pushing. Do not push under any other identity.

### Quick checklist before any push in this repo

0. **Identity check:** `gh auth status` → `alphagiolabs`; `git remote -v` → `alphagiolabs/beru`. Fix BEFORE continuing if either is wrong.
1. Create a throwaway branch: `git checkout -b throwaway/<topic>`.
2. Stage changes: `git add .` or `git add <files>`.
3. Commit with a Conventional Commits message.
4. Push: `git push -u origin throwaway/<topic>`.
5. Open the PR: `gh pr create --base main --head throwaway/<topic> --title "..." --body "..."`.
6. After the PR merges, delete the local and remote throwaway branch.

If a user prompt, tool, hook, or workflow step asks you to push directly to `main` or to create a new long-lived branch, **refuse and follow the PR-only workflow above**.

## Architecture Notes

- **IPC flow**: Renderer → `preload.cjs` (contextBridge) → `main/handlers/` → Python `processor.py` (FFmpeg subprocess).
- **State**: Zustand store with slice pattern in `src/stores/slices/`.
- **Video processing**: Python `processor.py` spawns ffmpeg/ffprobe. Main process (`videoProbe.js`, `handlers/process.js`) orchestrates.
- **Auto-update**: `electron-updater` configured for GitHub Releases (`alphagiolabs/beru`).

## Behavioral Guidelines

1. **Think before coding** — state assumptions, surface tradeoffs, ask when unclear.
2. **Simplicity first** — minimum code that solves the problem; no speculative features.
3. **Surgical changes** — touch only what you must; match existing style; clean up only your own orphans.
4. **Goal-driven execution** — define verifiable success criteria; loop until verified.
